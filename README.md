# 🌍 Global AQ Intelligence (Frontend)

![Dashboard Screenshot](../pow-eda-pipeline/plots/forecast_horizons.png)

> **An enterprise-grade, weather-aware PM2.5 forecasting engine frontend.**

This is the Next.js frontend for the **Global AQ Intelligence** platform. It serves statically generated JSON payloads produced by our autonomous Python ML pipeline, predicting air quality across 4 countries (USA, India, UK, Australia) using Gradient Boosting and physics-based thermodynamics.

`Next.js 15` | `TypeScript` | `Tailwind CSS` | `Recharts`

## ⚡ Architecture

This frontend is designed for **Zero-Runtime Dependency**. 
- Data is entirely static JSON served from `public/data/`.
- No API calls are made at runtime—everything is pre-generated by the backend prediction pipeline. 
- This is the optimal architecture for a forecasting product; the UI is blazingly fast, heavily cached on the Edge network, and completely immune to backend downtime.

## 🔬 The ML Backend (V7 Thermodynamics)

The heavy lifting is done in the companion `pow-eda-pipeline` repository. Our **V7 Direct Thermodynamics Engine** features:
- Independent models targeting specific horizons (`1d`, `7d`, `14d`, and `30d`).
- A physics-based **Weather-Weighted Interpolator** that applies thermodynamic logic (rain washouts, wind dispersion, stagnation spikes) using 16-day Open-Meteo forecasts.
- Strict chronological walk-forward validation (no temporal data leakage).

*For the complete MLOps architecture, feature engineering, and model training code, please see the [pow-eda-pipeline repository](../pow-eda-pipeline).*

## 🚀 Getting Started

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Run the development server:**
   ```bash
   npm run dev
   ```

3. Open [http://localhost:3000](http://localhost:3000) with your browser.

## 📊 Deployment

This project is configured for seamless deployment on Vercel. 
Because the backend Python pipeline automatically syncs its output JSONs directly into the `public/data` directory, deploying to Vercel requires zero complex build steps—it behaves exactly like a standard static Next.js application.
